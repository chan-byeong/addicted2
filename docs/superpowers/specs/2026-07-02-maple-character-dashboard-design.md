# 메이플 캐릭터 대시보드 설계

이 문서는 `/maple` 경로에 추가하는 메이플스토리 캐릭터 등록 및 조회
기능의 MVP 결정을 기록한다. 링크 아카이브와 같은 Next.js, Vercel,
Supabase 기반을 사용하되, 캐릭터 상세 정보는 NEXON Open API에서 서버를
통해 가져온다.

## 목표

사용자는 캐릭터 명을 입력해 공유 목록에 캐릭터를 등록한다. 앱은 캐릭터
명을 NEXON Open API의 `ocid`로 해석하고, 기본 정보를 Supabase에 저장한
뒤 카드 목록에 표시한다.

MVP는 다음 흐름에 집중한다.

- 누구나 등록된 캐릭터 목록을 읽을 수 있다.
- 누구나 캐릭터 명으로 캐릭터를 등록할 수 있다.
- 같은 `ocid`를 다시 등록하면 기존 행을 갱신한다.
- 카드에서 상세 페이지로 이동해 스탯과 장착 장비를 확인한다.
- 등록, 수정, 삭제용 비밀번호는 메이플 기능에는 적용하지 않는다.

## 화면 구성

`/maple` 화면은 캐릭터 명 입력 폼과 등록 캐릭터 카드 목록으로 구성한다.
카드는 왼쪽에 캐릭터 이미지, 오른쪽에 월드, 이름, 레벨, 직업, 최근 갱신
시각을 표시한다.

`/maple/[ocid]` 화면은 캐릭터 검사 화면이다. 상단에는 캐릭터 이미지와
기본 정보를 배치하고, 옆에는 주요 스탯 요약을 보여준다. 장비 영역은
아이콘 그리드와 선택된 장비 상세 패널로 구성한다.

시각 스타일은 기존 문서형 UI보다 조금 더 게임 도구에 가깝게 조정한다.
단, 모바일 웹뷰 사용성을 우선하며 amber와 emerald를 주요 색상으로 쓴다.

## 기술 구조

브라우저는 앱의 자체 API만 호출한다. NEXON Open API 키는
`NEXON_OPEN_API_KEY` 서버 환경 변수로만 보관한다.

API 경계는 다음과 같다.

- `GET /api/maple/characters`: 등록된 캐릭터 목록을 조회한다.
- `POST /api/maple/characters`: 캐릭터 명을 `ocid`로 해석하고 기본 정보를
  저장한다.
- `GET /api/maple/characters/[ocid]`: 기본 정보, 종합 스탯, 장착 장비를
  합쳐 반환한다.

Supabase에는 공유 목록에 필요한 최신 스냅샷만 저장한다. 상세 스탯과 장비
정보는 상세 페이지 진입 시 서버 route가 NEXON Open API에서 다시 가져온다.

## 데이터 모델

`maple_characters` 테이블은 공유 등록 목록을 보관한다. 상세 데이터는
저장하지 않는다.

필드는 다음과 같다.

- `id`: 기본 키
- `ocid`: NEXON 캐릭터 식별자, 고유 값
- `character_name`: 캐릭터 명
- `world_name`: 월드 명
- `character_class`: 직업
- `character_level`: 레벨
- `character_image`: 캐릭터 이미지 URL
- `created_at`: 생성 시각
- `updated_at`: 갱신 시각

## 결정 사항

이번 MVP에서 확정한 사항은 다음과 같다.

- 한국 메이플스토리 NEXON Open API를 사용한다.
- 등록은 공개로 두고 공용 비밀번호를 요구하지 않는다.
- 삭제와 수정 기능은 포함하지 않는다.
- 중복 등록은 `ocid` 기준 upsert로 처리한다.
- UI 검증은 수동 모바일 확인과 `lint`, `tsc`, `build`로 제한한다.
