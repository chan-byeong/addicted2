# 단톡방 링크 아카이브 설계

이 문서는 12명 규모의 단톡방 친구들이 모바일과 카카오톡 웹뷰에서
사용할 링크 아카이브 서비스의 MVP 설계를 정의한다. 서비스는 링크를
날짜별로 모으고 다시 찾는 데 집중하며, 회원가입 없이 읽기는 공개하고
쓰기 작업만 공용 비밀번호로 제한한다.

## 목표

이 서비스는 단톡방에서 흘러가는 유튜브, 쇼츠, 커뮤니티 글 링크를
날짜별로 보관하는 작은 아카이브다. 운영자와 참여자는 재미있거나 유익한
링크를 등록하고, 나중에 날짜나 검색어로 다시 찾을 수 있다.

- 사용자는 주로 모바일 환경과 카카오톡 웹뷰에서 접근한다.
- 읽기 접근은 누구나 가능하다.
- 링크 등록, 수정, 삭제는 공용 비밀번호를 아는 사람만 가능하다.
- 회원가입, 개인 계정, 댓글, 이모지 반응은 MVP에서 제외한다.
- 디자인은 나무위키처럼 단순한 문서형 UI를 지향한다.

## MVP 범위

MVP는 링크 중심 기능만 구현한다. 일정 기록, 사건 기록, 댓글, 반응,
투표, 개인별 프로필 기능은 초기 버전에 포함하지 않는다.

포함하는 기능은 다음과 같다.

- 날짜별 링크 목록 보기
- 최근 등록 링크 일부 보기
- 링크 등록, 수정, 삭제
- URL 메타데이터 자동 수집
- 메타데이터 수집 실패 시 수동 제목 입력
- 날짜 이동
- 텍스트 검색
- 링크 타입 필터

제외하는 기능은 다음과 같다.

- 회원가입과 로그인
- 댓글과 이모지 반응
- 일정과 사건 기록 전용 입력 화면
- 파일, 이미지, 영상 직접 업로드
- 관리자 전용 대시보드

## 사용자 흐름

사용자는 홈 화면에서 오늘 날짜의 링크 목록을 먼저 본다. 날짜 이동,
검색, 타입 필터를 사용해 과거 링크를 찾을 수 있다.

링크 등록 흐름은 다음과 같다.

1. 사용자가 **등록** 버튼을 누른다.
2. 사용자가 URL, 닉네임, 선택 메모, 공용 비밀번호를 입력한다.
3. 서버가 URL의 제목, 설명, 썸네일, 사이트명을 가져온다.
4. 메타데이터 수집에 성공하면 사용자가 미리보기를 확인한다.
5. 메타데이터 수집에 실패하면 사용자가 제목을 직접 입력한다.
6. 서버가 비밀번호를 검증한 뒤 링크를 저장한다.

수정과 삭제도 같은 공용 비밀번호를 요구한다. 삭제는 실제 행을 제거하지
않고 `deleted_at`을 채우는 소프트 삭제로 처리한다.

## 화면 구성

화면은 하나의 홈 화면을 중심으로 구성한다. 모바일에서 먼저 읽기 쉽고,
데스크톱에서는 폭이 너무 넓어지지 않도록 본문 너비를 제한한다.

홈 화면은 다음 영역으로 구성한다.

- 상단: 서비스 이름, 오늘 날짜, 등록 버튼
- 날짜 영역: 이전 날짜, 오늘, 다음 날짜 이동
- 필터 영역: 검색창과 타입 필터
- 날짜별 목록: 선택한 날짜의 링크 목록
- 최근 링크 영역: 날짜와 무관한 최근 등록 링크
- 등록 및 수정 모달: URL, 닉네임, 메모, 비밀번호, 미리보기

링크 항목은 다음 정보를 표시한다.

- 제목
- URL 또는 사이트명
- 타입
- 작성자 닉네임
- 선택 메모
- 등록 시간
- 썸네일

썸네일이 없거나 로드에 실패하면 이미지 영역을 숨기고 텍스트 정보만
표시한다.

## 기술 구조

서비스는 Vercel 배포를 전제로 한 Next.js 앱으로 구현한다. 데이터 저장은
Supabase Free 플랜의 Postgres를 사용한다.

주요 기술 선택은 다음과 같다.

- 프레임워크: Next.js App Router
- 배포: Vercel
- 데이터베이스: Supabase Postgres
- 쓰기 권한 검증: Next.js 서버 라우트 또는 서버 액션
- 비밀 값 관리: Vercel 환경 변수

브라우저는 공개 읽기 화면을 요청한다. 등록, 수정, 삭제 요청은 서버를
거치며, 서버가 공용 비밀번호를 검증한 뒤 Supabase에 쓰기 작업을 수행한다.
Supabase service role key는 클라이언트 번들에 포함하지 않는다.

## 데이터 모델

초기 데이터 모델은 `archive_items` 단일 테이블로 시작한다. 향후 일정이나
사건 기록을 추가할 수 있도록 `source_type`을 명시적으로 둔다.

`archive_items` 필드는 다음과 같다.

- `id`: 기본 키
- `url`: 원본 링크
- `title`: 링크 제목
- `description`: 링크 설명
- `image_url`: 썸네일 URL
- `site_name`: 사이트명
- `source_type`: `youtube`, `shorts`, `community`, `other`
- `note`: 작성자가 남긴 선택 메모
- `author_name`: 작성자가 직접 입력한 닉네임
- `entry_date`: 날짜별 일지 기준 날짜
- `created_at`: 생성 시각
- `updated_at`: 수정 시각
- `deleted_at`: 소프트 삭제 시각

조회 쿼리는 항상 `deleted_at IS NULL` 조건을 포함한다.

## API 설계

API는 읽기와 쓰기를 분리한다. 읽기 API는 공개로 두고, 쓰기 API는 서버에서
공용 비밀번호를 검증한다.

초기 API는 다음과 같다.

- `GET /api/items`: 날짜, 검색어, 타입 필터 기준으로 목록을 조회한다.
- `POST /api/items/metadata`: URL 미리보기 메타데이터를 조회한다.
- `POST /api/items`: 비밀번호 검증 후 링크를 등록한다.
- `PATCH /api/items/:id`: 비밀번호 검증 후 링크를 수정한다.
- `DELETE /api/items/:id`: 비밀번호 검증 후 링크를 소프트 삭제한다.

`POST /api/items/metadata`는 외부 사이트 접근 실패, 차단, 타임아웃을 정상
실패로 취급한다. 이 경우 클라이언트는 수동 제목 입력으로 전환한다.

## 링크 타입 추정

링크 타입은 처음에는 단순 규칙으로 추정한다. 완벽한 분류보다 등록 부담을
줄이는 것을 우선한다.

초기 규칙은 다음과 같다.

- `youtube.com/shorts`는 `shorts`로 분류한다.
- `youtube.com/watch`는 `youtube`로 분류한다.
- `youtu.be`는 메타데이터와 URL 패턴을 기준으로 `youtube` 또는 `shorts`로
  분류한다.
- 알려진 커뮤니티 도메인은 `community`로 분류한다.
- 그 외 도메인은 `other`로 분류한다.

사용자가 타입을 수정할 수 있게 만들면 자동 분류 실패를 보완할 수 있다.

## 권한과 보안

이 서비스는 사적인 단톡방용 도구이므로 복잡한 계정 체계 대신 공용
비밀번호를 사용한다. 읽기 접근은 열어두고, 데이터 변경 작업만 제한한다.

보안 규칙은 다음과 같다.

- 공용 비밀번호는 Vercel 환경 변수에 저장한다.
- 서버는 입력된 비밀번호와 환경 변수 값을 비교한다.
- 비밀번호가 틀리면 쓰기 작업을 수행하지 않는다.
- Supabase service role key는 서버에서만 사용한다.
- 클라이언트는 service role key를 알 수 없어야 한다.
- 삭제는 소프트 삭제로 처리한다.

공용 비밀번호 방식은 작은 단톡방에서는 운영 부담이 낮다. 다만 링크가 외부에
퍼질 가능성이 커지면 개인 계정이나 관리자 권한 체계로 전환해야 한다.

## 오류 처리

오류는 사용자가 입력을 잃지 않고 다시 시도할 수 있게 처리한다. 특히
카카오톡 웹뷰에서는 네트워크 상태가 불안정할 수 있으므로 짧고 명확한
메시지를 사용한다.

오류 처리 기준은 다음과 같다.

- 메타데이터 수집 실패: 수동 제목 입력으로 전환한다.
- 비밀번호 오류: 입력값을 유지하고 비밀번호 오류 메시지를 표시한다.
- 네트워크 오류: 화면 상단에 재시도 가능한 오류 메시지를 표시한다.
- Supabase 오류: 저장 실패 메시지를 표시하고 입력값을 유지한다.
- 썸네일 로드 실패: 이미지 영역을 숨긴다.

삭제는 사용자가 실수로 누르지 않도록 확인 단계를 둔다.

## 테스트 기준

MVP가 완료되려면 주요 사용자 흐름과 배포 설정이 검증되어야 한다.

완료 기준은 다음과 같다.

- 모바일 폭에서 날짜별 링크 목록을 읽기 쉽다.
- 카카오톡 웹뷰 크기에서 주요 버튼과 입력창이 사용 가능하다.
- URL 등록 시 제목과 썸네일이 자동으로 채워진다.
- 메타데이터 수집 실패 시 수동 제목으로 등록할 수 있다.
- 공용 비밀번호 없이는 등록, 수정, 삭제가 실패한다.
- 날짜 이동, 검색, 타입 필터가 함께 동작한다.
- Vercel 환경 변수만 설정하면 배포할 수 있다.

## 배포와 환경 변수

Vercel 프로젝트에는 Supabase 연결 정보와 공용 비밀번호를 환경 변수로
설정한다.

필요한 환경 변수는 다음과 같다.

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `ARCHIVE_WRITE_PASSWORD`

클라이언트에서 공개적으로 사용할 수 있는 값은 `NEXT_PUBLIC_` 접두사가 붙은
값으로 제한한다. 쓰기 작업과 관리자 권한에 필요한 값은 서버 전용 환경
변수로 둔다.

## 향후 확장

MVP가 실제 단톡방에서 쓰인 뒤 필요한 기능만 추가한다. 초기에는 입력 부담을
늘리는 기능을 피하고, 다시 찾기 기능의 품질을 먼저 확인한다.

확장 후보는 다음과 같다.

- 일정 기록과 사건 기록 타입 추가
- 태그 기능 추가
- 작성자 닉네임 자동 완성
- 월별 아카이브 보기
- 링크 중복 등록 감지
- 간단한 관리자 화면
- 개인 계정 기반 권한 체계

## 결정 사항

이번 설계에서 확정한 사항은 다음과 같다.

- Vercel, Next.js, Supabase Free 플랜을 사용한다.
- 읽기는 공개하고 쓰기만 공용 비밀번호로 제한한다.
- 작성자는 닉네임을 직접 입력한다.
- 기본 화면은 날짜별 일지형이며 최근 링크 일부를 함께 보여준다.
- MVP는 링크 중심으로 구현한다.
- 링크 메타데이터는 서버에서 자동 수집한다.
- 찾기 기능은 날짜 이동, 텍스트 검색, 타입 필터까지만 제공한다.
- 댓글과 이모지 반응은 MVP에서 제외한다.
