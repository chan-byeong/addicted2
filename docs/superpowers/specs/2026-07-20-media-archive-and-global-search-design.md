# 미디어 아카이브 및 전체 검색 설계

이 문서는 기존 링크 아카이브에 사진과 동영상 업로드를 추가하고, 검색과
타입 필터의 범위를 선택 날짜가 아닌 전체 아카이브로 확장하는 결정을
정의한다. 날짜별 일지 화면은 기본 탐색 방식으로 유지한다.

## 범위

아카이브 항목은 링크, 사진, 동영상 중 하나다. 사진과 동영상은 링크와
동일하게 날짜, 작성자, 선택 메모를 가진다.

- 링크는 기존 URL 메타데이터 수집 흐름을 유지한다.
- 사진은 JPG, PNG, GIF, WebP, AVIF, HEIC 형식을 지원한다.
- 동영상은 MP4, MOV, WebM 형식을 지원한다.
- 사진은 10MB, 동영상은 50MB로 제한한다.
- 미디어 파일은 공개 읽기가 가능한 Supabase Storage 버킷에 저장한다.

## 업로드 흐름

파일은 Vercel 함수의 요청 본문을 통과하지 않는다. 서버는 제한된 업로드
권한만 담은 signed upload token을 발급하고, 브라우저는 Supabase Storage로
직접 전송한다.

1. 사용자가 **등록**에서 **사진** 또는 **동영상**을 선택한다.
2. 클라이언트가 파일 형식과 크기를 확인한다.
3. `POST /api/items/media-upload`가 임의 저장 경로와 signed token을 발급한다.
4. 클라이언트가 파일을 Supabase Storage에 직접 업로드한다.
5. `POST /api/items`가 업로드된 객체를 확인하고 아카이브 행을 생성한다.

업로드가 완료된 뒤 행 생성에 실패하면 서버는 해당 Storage 객체를
삭제한다. 사용자가 최종 저장 전에 페이지를 닫아 남은 객체를 정리하는
주기 작업은 이번 범위에 포함하지 않는다.

## 데이터 모델

기존 `archive_items` 테이블을 유지하고 미디어 식별 필드를 추가한다. 기존
행은 자동으로 `link` 타입이 된다.

- `content_type`: `link`, `image`, `video`
- `storage_path`: Supabase Storage 내부 객체 경로
- `media_mime_type`: 업로드 파일 MIME 타입
- `media_size`: 업로드 파일 크기(바이트)
- `search_text`: 전체 검색에 사용하는 생성 열

공개 미디어는 `archive-media` 버킷에 저장한다. 업로드, 이동, 삭제는 signed
token 또는 서버의 service role을 통해서만 수행한다.

## 조회와 검색

필터가 없는 기본 화면은 선택 날짜의 항목만 표시한다. 검색어나 타입
필터가 활성화되면 날짜 조건을 제거하고 전체 아카이브에서 결과를 찾는다.

- 텍스트 검색은 제목, 설명, 메모, URL, 사이트명을 대상으로 한다.
- 링크 필터는 유튜브, 쇼츠, 커뮤니티, 기타를 유지한다.
- 미디어 필터는 사진과 동영상을 추가한다.
- 전체 검색 결과는 최신 등록 순으로 최대 50개를 표시한다.
- 검색과 타입 필터는 함께 적용할 수 있다.

## 카드 표시

카드는 항목 타입에 맞는 내용을 표시한다. 링크 설명 본문은 목록에서
표시하지 않는다.

- 링크 카드는 썸네일, 제목 링크, 사이트, 타입, 작성자, 시간을 표시한다.
- 사진 카드는 업로드 이미지, 파일명, 작성자, 시간을 표시한다.
- 동영상 카드는 재생 컨트롤, 파일명, 작성자, 시간을 표시한다.
- 긴 메모는 두 줄로 접고 카드 토글을 사용해 전체 내용을 표시한다.
- 링크 제목은 카드 토글과 별개로 원본 URL을 연다.

## 검증 기준

구현 완료 여부는 실제 저장과 전체 검색 범위를 기준으로 확인한다.

- 지원되는 사진과 동영상을 메모와 함께 등록할 수 있다.
- 제한을 넘거나 타입이 맞지 않는 파일은 업로드 전에 거절된다.
- 선택 날짜와 다른 날짜의 항목도 검색 결과에 포함된다.
- 사진 또는 동영상 필터가 모든 날짜의 항목을 조회한다.
- 필터를 해제하면 선택 날짜의 일지 화면으로 돌아간다.
- 모바일 화면에서 이미지와 동영상 컨트롤을 사용할 수 있다.
